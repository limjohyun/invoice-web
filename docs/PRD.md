# Notion Invoice Web MVP PRD

## 🎯 핵심 정보

**목적**: 별도의 서버/DB 구축 없이 Notion 데이터베이스를 백엔드로 활용해 견적서를 손쉽게 작성·관리·공유하는 문제를 해결한다.
**사용자**: 이미 Notion으로 업무를 정리하고 있는 1인 사업가, 프리랜서, 소규모 비즈니스 담당자.

## 🚶 사용자 여정

```

1. 홈 페이지
   ↓ 로그인 버튼 클릭

2. 로그인 페이지 / 회원가입 페이지
   ↓ 인증 성공

   [Notion 미연동 사용자] → Notion 연동 설정 페이지 → 연동 완료 →
   [Notion 연동 완료 사용자] ─────────────────────────→ 대시보드
   ↓

3. 대시보드 (견적서 목록)
   ↓ 새 견적서 작성 클릭        ↓ 견적서 항목 클릭        ↓ 사이드 메뉴 이동

4. 견적서 작성 페이지          5. 견적서 상세 페이지      6. 거래처 관리 페이지 / 템플릿 관리 페이지
   ↓ 저장 (Notion 동기화)         ↓ 수정 클릭 → 견적서 작성 페이지(수정 모드)
   └──────────────────────────→ 5. 견적서 상세 페이지
                                   ↓ PDF 다운로드 / 삭제

7. 완료 → 대시보드로 복귀 또는 새 견적서 작성

```

## ⚡ 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F001** | 견적서 목록 조회 | Notion 데이터베이스에서 견적서 목록을 불러와 표시 | 핵심 데이터 확인 기능 | 대시보드 |
| **F002** | 견적서 작성 | 품목, 수량, 단가, 거래처 등을 입력해 신규 견적서 생성 | 서비스의 핵심 가치 | 견적서 작성 페이지 |
| **F003** | 견적서 상세 조회 | 특정 견적서의 전체 내용을 확인 | 작성 내용 검토 필수 | 견적서 상세 페이지 |
| **F004** | 견적서 수정 | 기존 견적서 내용을 변경하고 Notion에 반영 | 견적 수정 요청 대응 | 견적서 작성 페이지, 견적서 상세 페이지 |
| **F005** | 견적서 삭제 | 불필요한 견적서를 목록/Notion에서 제거 | 데이터 정리 필수 | 대시보드, 견적서 상세 페이지 |
| **F006** | Notion 데이터베이스 동기화 | 견적서 생성/수정/삭제 시 Notion API로 실시간 반영 | Notion을 백엔드로 사용하는 핵심 로직 | Notion 연동 설정 페이지, 대시보드, 견적서 작성 페이지, 견적서 상세 페이지 |
| **F007** | 견적서 PDF 다운로드 | 견적서 상세 내용을 PDF 파일로 내보내기 | 거래처 전달을 위한 필수 산출물 | 견적서 상세 페이지 |

### 2. MVP 필수 지원 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F010** | 기본 인증 | 회원가입/로그인/로그아웃만 지원 | 서비스 이용을 위한 최소 인증 | 로그인 페이지, 회원가입 페이지 |
| **F011** | Notion 연동 설정 | Notion Integration Token 및 대상 데이터베이스 등록 | Notion 백엔드 연동의 전제 조건 | Notion 연동 설정 페이지 |
| **F012** | 거래처 관리 | 거래처명, 담당자, 연락처 등 최소 정보 등록 및 조회 | 견적서 작성 시 거래처 선택 필요 | 거래처 관리 페이지, 견적서 작성 페이지 |

## ✅ 정합성 검증 결과

- 기능 명세(F001~F013) 전 항목이 페이지별 상세 기능에 매핑되어 구현 페이지가 명시됨
- 메뉴 구조의 모든 메뉴 항목(홈, 로그인, 회원가입, 대시보드, 견적서 작성, 거래처 관리, 템플릿 관리, Notion 연동 설정, 로그아웃)이 페이지별 상세 기능에 페이지로 존재함
- 페이지별 상세 기능에 명시된 모든 구현 기능 ID가 기능 명세 표에 정의되어 있음
- 기능 명세에만 존재하고 페이지에서 구현되지 않은 기능 없음, 페이지에만 존재하고 기능 명세에 없는 기능 없음
